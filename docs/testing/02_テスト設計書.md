## 2. テスト戦略概要

### 2.1 スキル管理システム特化テスト方針

**データ品質・ユーザビリティ重視**のテスト戦略を採用し、以下を重点的に検証：

1. **スキル評価の正確性**: 4段階評価（×/△/○/◎）の適切な管理
2. **セキュリティ強化**: 認証・認可・データ保護の徹底検証
3. **パフォーマンス保証**: レスポンス時間1秒以内の達成
4. **運用品質**: 自動化・監視・復旧の信頼性

### 2.2 テストツールスタック

| テストレベル | ツール | 目的 | カバレッジ目標 |
|-------------|--------|------|---------------|
| **ユニットテスト** | Vitest + @testing-library/react | コンポーネント・関数の単体テスト | 90%以上 |
| **統合テスト** | Vitest + MSW + Supertest | API・DB連携テスト | 80%以上 |
| **E2Eテスト** | Playwright 1.56+ | ユーザーシナリオテスト | 主要フロー100% |
| **AI駆動E2Eテスト** | Playwright Agents (Planner/Generator/Healer) | テスト自動生成・自動修復 | 新規テスト100% AI生成 |
| **パフォーマンステスト** | k6 + Lighthouse CI | 負荷・性能テスト | 要件100%達成 |
| **セキュリティテスト** | OWASP ZAP + axe-core | 脆弱性・アクセシビリティ | 脆弱性0件 |

### 2.3 Playwright Agents による革新的テスト戦略

#### 2.3.1 AI駆動テスト自動化の導入

**Playwright Agents（Playwright 1.56+）** を活用し、テスト計画の生成からテストコードの実装、失敗時の自動修復までを AI が支援する革新的なアプローチを採用します。

#### 2.3.2 3つのエージェントの役割

| エージェント | 役割 | 効果 |
|------------|------|------|
| **Planner** | ユーザーストーリーからテスト計画を自動生成 | テスト設計工数を87.5%削減 |
| **Generator** | テスト計画から実行可能なコードを自動生成 | テスト実装工数を87.5%削減 |
| **Healer** | テスト失敗を分析し自動修復 | メンテナンス工数を80%削減 |

#### 2.3.3 導入メリット

- **工数削減**: テスト作成・保守の大幅な効率化（年間200時間削減見込み）
- **品質向上**: AIによる包括的なテストシナリオ生成
- **セルフヒーリング**: UI変更時の自動テスト修復
- **知識共有**: AI生成テストがベストプラクティスのテンプレートとなる

## 3. CI/CD統合

### 3.1 GitHub Actions設定（Playwright Agents統合）

```yaml
# .github/workflows/test.yml
name: Test Suite
on:
  push:
    branches: [main, develop]
  pull_request:
    branches: [main, develop]

jobs:
  test:
    runs-on: ubuntu-latest

    services:
      postgres:
        image: postgres:15
        env:
          POSTGRES_PASSWORD: postgres
          POSTGRES_DB: test_db
        options: >-
          --health-cmd pg_isready
          --health-interval 10s
          --health-timeout 5s
          --health-retries 5
        ports:
          - 5432:5432

    steps:
      - uses: actions/checkout@v4

      - uses: actions/setup-node@v4
        with:
          node-version: '18'
          cache: 'npm'

      - name: Install dependencies
        run: npm ci

      - name: Setup test database
        run: |
          npx prisma migrate deploy
          npx prisma db seed
        env:
          DATABASE_URL: postgresql://postgres:postgres@localhost:5432/test_db

      - name: Type check
        run: npm run type-check

      - name: Lint
        run: npm run lint

      - name: Unit tests
        run: npm run test:unit
        env:
          DATABASE_URL: postgresql://postgres:postgres@localhost:5432/test_db

      - name: Integration tests
        run: npm run test:integration
        env:
          DATABASE_URL: postgresql://postgres:postgres@localhost:5432/test_db

      - name: Build application
        run: npm run build

      - name: E2E tests with Playwright Agents (Auto-Healing)
        run: npx playwright agent healer --auto-fix --test tests/e2e/ --max-attempts=3
        env:
          DATABASE_URL: postgresql://postgres:postgres@localhost:5432/test_db
          ANTHROPIC_API_KEY: ${{ secrets.ANTHROPIC_API_KEY }}

      - name: Upload coverage
        uses: codecov/codecov-action@v3
        with:
          file: ./coverage/lcov.info

      - name: Upload test results
        uses: actions/upload-artifact@v3
        if: always()
        with:
          name: test-results
          path: |
            test-results/
            coverage/
            playwright-report/
```

### 3.2 Playwright Agents CI/CD ワークフロー

#### 3.2.1 PRベースのテスト計画自動生成

```yaml
# .github/workflows/pr-test-plan.yml
name: Generate Test Plan for PR

on:
  pull_request:
    types: [opened, synchronize]

jobs:
  generate-test-plan:
    runs-on: ubuntu-latest

    steps:
      - uses: actions/checkout@v4

      - uses: actions/setup-node@v4
        with:
          node-version: '18'

      - name: Install dependencies
        run: npm ci

      - name: Generate test plan from PR description
        run: |
          npx playwright agent planner \
            --story "${{ github.event.pull_request.body }}" \
            --output tests/plans/pr-${{ github.event.pull_request.number }}.md
        env:
          ANTHROPIC_API_KEY: ${{ secrets.ANTHROPIC_API_KEY }}

      - name: Generate test code from plan
        run: |
          npx playwright agent generator \
            --plan tests/plans/pr-${{ github.event.pull_request.number }}.md \
            --output tests/e2e/pr-${{ github.event.pull_request.number }}.spec.ts
        env:
          ANTHROPIC_API_KEY: ${{ secrets.ANTHROPIC_API_KEY }}

      - name: Comment test plan and code on PR
        uses: actions/github-script@v6
        with:
          script: |
            const fs = require('fs')
            const plan = fs.readFileSync('tests/plans/pr-${{ github.event.pull_request.number }}.md', 'utf8')
            const code = fs.readFileSync('tests/e2e/pr-${{ github.event.pull_request.number }}.spec.ts', 'utf8')

            github.rest.issues.createComment({
              issue_number: context.issue.number,
              owner: context.repo.owner,
              repo: context.repo.repo,
              body: `## AI生成テスト計画とコード\n\n### テスト計画\n\n${plan}\n\n### 生成されたテストコード\n\n\`\`\`typescript\n${code}\n\`\`\``
            })
```

### 4.2 テストデータベース管理

```typescript
// src/test/database.ts
import { PrismaClient } from '@prisma/client'
import { execSync } from 'child_process'

export async function createTestDatabase(): Promise<PrismaClient> {
  const testDbUrl = process.env.TEST_DATABASE_URL || 'postgresql://postgres:postgres@localhost:5432/test_db'
  
  // テストDB初期化
  execSync('npx prisma migrate reset --force', {
    env: { ...process.env, DATABASE_URL: testDbUrl }
  })
  
  const prisma = new PrismaClient({
    datasources: {
      db: { url: testDbUrl }
    }
  })
  
  await prisma.$connect()
  return prisma
}

export async function cleanupTestDatabase(prisma: PrismaClient): Promise<void> {
  // テーブルクリーンアップ（外部キー制約順序に注意）
  await prisma.skill.deleteMany()
  await prisma.careerGoal.deleteMany()
  await prisma.workRecord.deleteMany()
  await prisma.trainingRecord.deleteMany()
  await prisma.user.deleteMany()
  
  await prisma.$disconnect()
}

export async function seedTestData(prisma: PrismaClient) {
  // テストデータ投入
  const user = await prisma.user.create({
    data: {
      email: 'test@example.com',
      name: 'テストユーザー',
      empNo: 'EMP001',
      department: '開発部',
      role: 'user'
    }
  })
  
  const skill = await prisma.skill.create({
    data: {
      name: 'JavaScript',
      category: 'プログラミング',
      level: '○',
      userId: user.id
    }
  })
  
  return { user, skill }
}
```
